# Architecture Details

The CTNSG framework achieves zero-hallucination and $\mathcal{O}(1)$ latency through rigorous mathematical constraints across its modules.

## Mathematical Constraints

### FandE Score Optimization
Unlike standard Triple F1 used in classic NLP pipelines, the **Features and Edges (FandE) Score** prevents representational saturation. It ensures the graph diffusion process maintains genuine topological reasoning rather than collapsing into trivial star graphs.

### Magnetic Laplacian Positional Encodings (mLPE)
Because the discourse graph is directed (causal logic flows one way), standard graph positional encodings fail. mLPE preserves directionality in the spatial representations passed to the Graph VQ-Transformer.

### Probabilistic Sentential Decision Diagrams (PSDD)
Domain schemas and hard constraints are compiled entirely offline into PSDDs. This forms a mathematically guaranteed semantic prior. During inference, Projected Gradient Descent (PGD) convex optimization allows the model to traverse this circuit to enforce soft constraints (fairness, sentiment) without breaking hard logical bounds.

### Virtual Node Pooling (VNPool) & LoRA
The discrete macroplan generated by Module 1 is projected into the continuous embedding space of the Base LLM (Module 3) using VNPool. To prevent the LLM from suffering optimization collapse during this soft-prompting, the continuous vectors are stabilized by injecting LoRA adapters specifically trained for graph-to-text alignment.
