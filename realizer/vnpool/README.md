# Module 3: Virtual Node Pool (VNPool) Realizer

This module acts as the "Realizer" bridge between the discrete graph topologies generated by the Macroplanner and the continuous semantic space of the **Qwen-3.5** Large Language Model. It compresses and injects the graph as a dynamic soft prompt, steering the LLM's logic without overwhelming its context window.

## Key Components

### 1. `decoder.py` (Hierarchical RVQ Recombination)
Takes the discrete `[batch_size, num_nodes, N=4]` tensor generated by RelDiT. Instead of a standard single lookup, it executes parallel lookups across 4 distinct `nn.Embedding` codebooks and **sums** them together. This preserves the hierarchical semantic fidelity of the Residual Vector Quantization, restoring the continuous `[batch_size, num_nodes, 256]` graph.

### 2. `pooling.py` (Pure Perceiver IO Bottleneck)
Variable-sized graphs must be compressed to a fixed length to act as stable soft prompts. We utilize a **Perceiver IO Cross-Attention Bottleneck** with exactly $K=8$ latent queries.
- **Strict Information Bottleneck:** The decoded graph nodes are processed exclusively as a permutation-invariant set of Keys and Values. There is explicitly zero self-attention or GNN message passing amongst the graph nodes within the pooler.

### 3. `projector.py` (LLM Projector)
Adapts the compressed $D_{GVT}$ representations to the $D_{LLM}$ dimension using an empirically validated `Linear -> ReLU -> Linear` Multi-Layer Perceptron.

### 4. `injector.py` (Textualization Scaffold)
Relying purely on soft prompts causes catastrophic hallucination of opaque entities. This injector utilizes a **Textualization Scaffold**, strictly concatenating the prompt in the order: `[8_VN_Tokens, Textualized_Graph_Embeddings, User_Query]`. This anchors the abstract topological embeddings to explicit semantic triples.

### 5. `model.py` (Strict Calibrated LoRA Realizer)
The master wrapper that connects the pipeline. Crucially, soft prompt tuning on dense graph topologies suffers from optimization collapse if the LLM backbone remains entirely frozen. 
- The `peft` library is a **strict requirement**. This script injects **Low-Rank Adaptation (LoRA)** adapters into the Qwen LLM's `q_proj` and `v_proj` layers, keeping the adapters actively trainable during projection.
- It enforces strict grid-searched calibration: **Rank $r=16$**, **$\alpha=32$**, and **Dropout $0.05$**, ensuring stable gradient flow directly from the LLM outputs all the way back into the Perceiver IO pooler.
